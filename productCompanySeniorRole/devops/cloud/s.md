# THE STORY OF CLOUD (AWS) — HOW COMPUTERS LEFT BUILDINGS AND ENTERED THE SKY

*A journey from buying servers → renting global infrastructure*

---

## PROLOGUE — THE COMPANY THAT WAITED 3 MONTHS FOR A SERVER

A startup wanted to launch an app.

They needed a server.

Steps:

```
buy hardware
wait delivery
install OS
configure network
setup security
deploy app
```

Total time:

> 2–3 months.

Problem?

By the time server was ready…

Market opportunity was gone.

Engineers realized:

> Infrastructure should be instant.

That realization created:

> Cloud Computing.

---

# CHAPTER 1 — THE OLD WORLD (BEFORE CLOUD)

Before cloud, companies ran:

> on-premise data centers.

Meaning:

```
buy servers
buy racks
buy cooling
buy electricity
hire admins
maintain hardware
replace failures
```

Infrastructure was:

```
expensive
slow
inflexible
risky
```

Scaling required buying new hardware.

Not practical.

---

# CHAPTER 2 — THE REAL PROBLEM

Companies didn’t actually want servers.

They wanted:

> computing power.

They didn’t care where machine existed.

They only cared:

```
CPU
memory
storage
network
availability
```

So engineers asked:

> What if infrastructure was available instantly on demand?

---

# CHAPTER 3 — THE BREAKTHROUGH IDEA

Instead of buying servers…

What if you could:

> rent infrastructure like electricity?

Use when needed.

Pay only for usage.

This idea created:

> Cloud.

---

# CHAPTER 4 — WHAT CLOUD REALLY IS

Cloud is:

> remote infrastructure accessible over internet.

Instead of:

```
your server in your office
```

You get:

```
virtual server in remote data center
```

You access it through network.

---

# CHAPTER 5 — WHY AWS CHANGED EVERYTHING

AWS introduced:

> Infrastructure as a Service.

Meaning:

You can launch server in seconds:

```
click → server ready
```

No waiting.

No hardware.

No setup.

Just instant infrastructure.

---

# CHAPTER 6 — THE MOST IMPORTANT REALIZATION

Cloud is not someone else’s computer.

Cloud is:

> programmable infrastructure.

You can create servers using code.

Scale using code.

Delete using code.

Infrastructure becomes software.

Everything needed to run systems.

Cloud is complete platform.
