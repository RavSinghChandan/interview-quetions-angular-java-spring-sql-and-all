# CENTRALIZED STATE ARCHITECTURE

State management libraries create a **single source of truth**.

Architecture:

```text
Component
    ↓
Action dispatched
    ↓
Reducer updates state
    ↓
Store updated
    ↓
Components receive new state
```

This architecture ensures:

• predictable state changes
• easier debugging
• better scalability

==================================================

# NGRX ARCHITECTURE (ENTERPRISE STANDARD)

NgRx is Angular’s most common state management library.

NgRx architecture follows **Redux principles**.

Main components:

```text
Actions
Reducers
Store
Selectors
Effects
```

==================================================

# ACTIONS

Actions represent **events in the application**.

Example action:

```ts
import { createAction } from '@ngrx/store';

export const loadUsers = createAction('[User] Load Users');
```

Actions are dispatched by components.

Example:

```ts
this.store.dispatch(loadUsers());
```

### Mental Anchor

> Actions describe **what happened in the system**.

==================================================

# REDUCERS

Reducers define **how state changes**.

Example reducer:

```ts
import { createReducer, on } from '@ngrx/store';

const initialState = {
 users: []
};

export const userReducer = createReducer(
 initialState,
 on(loadUsersSuccess, (state, { users }) => ({
   ...state,
   users
 }))
);
```

Reducers must be:

• pure functions
• predictable
• immutable

==================================================

# STORE

The store holds the **entire application state**.

Example:

```text
Store
 ├── userState
 ├── authState
 └── tradeState
```

Components access state using selectors.

Example:

```ts
this.store.select(selectUsers)
```

==================================================

# SELECTORS

Selectors retrieve data from the store.

Example:

```ts
export const selectUsers = createSelector(
 selectUserState,
 state => state.users
);
```

Usage in component:

```ts
users$ = this.store.select(selectUsers);
```

Selectors improve:

• performance
• reusability
• separation of concerns

==================================================

# EFFECTS (ASYNC OPERATIONS)

Reducers must be pure.

Async tasks like HTTP calls are handled using **effects**.

Example effect:

```ts
loadUsers$ = createEffect(() =>
 this.actions$.pipe(
   ofType(loadUsers),
   switchMap(() =>
     this.userService.getUsers()
       .pipe(
         map(users => loadUsersSuccess({ users }))
       )
   )
 )
);
```

Flow:

```text
Action dispatched
     ↓
Effect triggers API call
     ↓
Result dispatched as new action
     ↓
Reducer updates state
```

# SIGNAL-BASED STATE (MODERN ANGULAR)

Angular introduced **signals** for reactive state.

Example:

```ts
import { signal } from '@angular/core';

users = signal([]);
```

Update state:

```ts
this.users.set(newUsers);
```

Access value:

```ts
this.users()
```

Signals automatically trigger UI updates.

Benefits:

• simpler than NgRx
• predictable updates
• better performance

==================================================

# SERVICE-BASED STATE MANAGEMENT

For medium-size applications, services with RxJS are often enough.

Example:

```ts
@Injectable({ providedIn: 'root' })
export class UserStateService {

 private usersSubject = new BehaviorSubject([]);
 users$ = this.usersSubject.asObservable();

 setUsers(users) {
   this.usersSubject.next(users);
 }

}
```

Components subscribe:

```ts
this.userState.users$.subscribe(users => {
 this.users = users;
});
```

# PERFORMANCE CONSIDERATIONS

State updates trigger UI updates.

Best practices:

• avoid large global state objects
• use selectors for efficient updates
• keep reducers pure
• use immutability

Example immutable update:

```ts
return {
 ...state,
 users: [...state.users, newUser]
};
```
