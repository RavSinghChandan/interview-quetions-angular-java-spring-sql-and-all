# CORE IDEA: ANGULAR IS A COMPILED FRAMEWORK

Angular is not interpreted at runtime like traditional JavaScript frameworks.

Angular **compiles templates into JavaScript instructions**.

Example template:

```html
<h1>{{title}}</h1>
```

Angular compiler transforms this into:

```javascript
ɵɵelementStart(0, "h1");
ɵɵtext(1);
ɵɵelementEnd();
ɵɵtextBinding(1, ctx.title);
```

These instructions are executed by Angular's **rendering engine**.

### Mental Anchor

> Angular templates are compiled into **efficient rendering instructions**.

# ANGULAR BUILD PIPELINE

When developers run:

```bash
ng build
```

Angular executes a complex build pipeline.

Pipeline:

```text
TypeScript Compilation
        ↓
Angular Template Compilation
        ↓
Bundling (Webpack / Build Tools)
        ↓
Tree Shaking
        ↓
Minification
        ↓
Optimized JavaScript Bundles
```

Output:

```text
main.js
runtime.js
polyfills.js
styles.css
```

These bundles are served to the browser.

# TYPESCRIPT COMPILATION

Angular applications are written in **TypeScript**.

Example:

```ts
export class UserComponent {
 title = "Users";
}
```

TypeScript compiler converts it to JavaScript.

Example:

```javascript
class UserComponent {
 constructor() {
   this.title = "Users";
 }
}
```

But Angular adds an additional compilation step for **templates and metadata**.

==================================================

# ANGULAR TEMPLATE COMPILER

Angular compiler processes templates.

It analyzes:

• template HTML
• component metadata
• data bindings
• directives

Example template:

```html
<div *ngIf="isAdmin">
 Admin Panel
</div>
```

Angular converts it into optimized **view creation instructions**.

This eliminates runtime template parsing.

# JIT VS AOT COMPILATION

Angular supports two compilation modes.

---

# JIT (JUST-IN-TIME)

Compilation happens in the browser at runtime.

Development mode:

```bash
ng serve
```

Process:

```text
Browser loads application
       ↓
Angular compiler runs in browser
       ↓
Templates compiled at runtime
```

Advantages:

• faster development
• easier debugging

Disadvantages:

• larger bundle size
• slower startup

---

# AOT (AHEAD-OF-TIME)

Compilation happens during build.

Production build:

```bash
ng build --configuration production
```

Process:

```text
Templates compiled during build
       ↓
Optimized JavaScript generated
       ↓
Browser executes compiled code
```

Advantages:

• smaller bundles
• faster startup
• improved security

### Mental Anchor

> Production Angular apps always use **AOT compilation**.

# THE IVY RENDERING ENGINE

Angular introduced **Ivy** as its new rendering engine.

Before Ivy, Angular used the **View Engine**.

Ivy improves:

• bundle size
• compilation speed
• runtime performance

Example Ivy instruction:

```javascript
ɵɵelementStart
ɵɵelementEnd
ɵɵtextBinding
```

These are extremely lightweight operations.

==================================================

# BENEFITS OF IVY

Ivy introduced several improvements.

```text
Smaller bundle sizes
Better tree shaking
Faster compilation
Improved debugging
Standalone components
```

Ivy also allows:

• partial compilation
• incremental builds

==================================================

# TREE SHAKING

Tree shaking removes unused code.

Example:

```ts
import { map } from 'rxjs/operators';
```

If the operator is never used:

Angular removes it from the bundle.

Benefits:

• smaller JavaScript bundles
• faster application loading

==================================================

# CODE SPLITTING

Angular splits bundles into multiple files.

Example:

```text
main.js
vendor.js
runtime.js
lazy-module.js
```

Lazy-loaded modules create additional bundles.

Example:

```ts
loadChildren: () =>
 import('./admin/admin.module')
```

Admin module becomes its own bundle.

==================================================

# MINIFICATION

During production build:

Angular removes unnecessary characters.

Example:

Original code:

```javascript
function calculateTotal(price, quantity) {
 return price * quantity;
}
```

Minified code:

```javascript
function a(b,c){return b*c}
```

This reduces file size.

==================================================

# BUILD OPTIMIZATION TECHNIQUES

Angular build system optimizes code using:

Dead code elimination

# SOURCE MAPS

During development, Angular provides **source maps**.

These allow debugging original TypeScript code even though the browser runs compiled JavaScript.

Example:

```text
TypeScript file
      ↓
Compiled JS file
      ↓
Source map maps them together
```
