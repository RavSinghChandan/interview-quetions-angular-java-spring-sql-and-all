# MEMORY LEAK DETECTION

Angular applications may leak memory if subscriptions are not cleaned up.

Example problem:

```ts id="obs6"
this.userService.user$
 .subscribe(user => {
   this.user = user;
 });
```

If component is destroyed but subscription remains active:

Memory leak occurs.

Solution:

```ts id="obs7"
takeUntil(this.destroy$)
```

Or use:

```html id="obs8"
async pipe
```

The async pipe automatically unsubscribes.

# GLOBAL ERROR HANDLING

Angular allows centralized error handling using `ErrorHandler`.

Example:

```ts id="obs11"
@Injectable()
export class GlobalErrorHandler implements ErrorHandler {

 handleError(error: any) {
   console.error('Global Error:', error);
 }

}
```

Register in module:

```ts id="obs12"
providers: [
 { provide: ErrorHandler, useClass: GlobalErrorHandler }
]
```

This ensures all application errors are captured.

# API MONITORING

Frontend applications depend heavily on APIs.

Monitoring tools track:

• API response time
• failed requests
• retry attempts

Example logging interceptor:

```ts id="obs13"
intercept(req: HttpRequest<any>, next: HttpHandler) {

 const start = Date.now();

 return next.handle(req).pipe(
   tap(() => {
     const duration = Date.now() - start;
     console.log(req.url, duration);
   })
 );

}
```
