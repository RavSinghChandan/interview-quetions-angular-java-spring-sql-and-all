# CORE IDEA: WHAT A COMPONENT REALLY IS

Most developers think a component is:

❌ Just a UI block

But a component is actually:

```text
State + Template + Behavior + Lifecycle
```

Angular components encapsulate:

• UI structure
• state management
• user interaction logic
• rendering behavior

Example:

```ts
@Component({
 selector: 'app-user',
 template: `<h2>{{user.name}}</h2>`
})
export class UserComponent {
 user = { name: "Chandan" };
}
```

### Mental Anchor

> A component is a **self-contained UI unit with state and lifecycle**.

# COMPONENT STRUCTURE (ANATOMY)

Every Angular component has three major parts.

```
Component
│
├── Template
├── Class
└── Metadata
```

Example:

```ts
@Component({
 selector: 'app-dashboard',
 templateUrl: './dashboard.component.html',
 styleUrls: ['./dashboard.component.css']
})
export class DashboardComponent {
 title = "Dashboard";
}
```

### Breakdown

**Metadata**

Defines Angular behavior.

```
selector
template
styles
changeDetection
providers
```

**Class**

Contains:

• state
• methods
• lifecycle hooks

**Template**

Defines UI layout and bindings.

# COMPONENT TREE (APPLICATION STRUCTURE)

Angular applications are **component trees**.

Example structure:

```
AppComponent
│
├── NavbarComponent
│
├── DashboardComponent
│   ├── UserWidgetComponent
│   └── ChartWidgetComponent
│
└── FooterComponent
```

Angular renders the entire application through this tree.

Each component manages:

• its own template
• its own data
• its own dependencies

### Mental Anchor

> Angular renders UI as a **hierarchical component tree**.

==================================================

# DATA FLOW IN COMPONENT TREES

Angular enforces **unidirectional data flow**.

```
Parent Component
        ↓
   Child Component
```

Data flows downward through:

```ts
@Input()
```

Example:

```ts
@Component({
 selector: 'app-user',
 template: `<h2>{{user.name}}</h2>`
})
export class UserComponent {

 @Input()
 user!: any;

}
```

Parent usage:

```html
<app-user [user]="currentUser"></app-user>
```

### Mental Anchor

> Parent provides data. Child renders it.

==================================================

# CHILD TO PARENT COMMUNICATION

Child components emit events upward.

Example:

```ts
@Output()
userUpdated = new EventEmitter<User>();
```

Emit event:

```ts
this.userUpdated.emit(updatedUser);
```

Parent listens:

```html
<app-user (userUpdated)="onUserUpdated($event)">
</app-user>
```

This creates a **controlled event flow**.

# SMART VS DUMB COMPONENTS (ENTERPRISE DESIGN)

Large Angular systems use two types of components.

---

# SMART COMPONENTS (CONTAINER)

Responsibilities:

• fetch data
• manage state
• communicate with services

Example:

```
DashboardContainerComponent
```

---

# DUMB COMPONENTS (PRESENTATIONAL)

Responsibilities:

• display UI
• receive data through inputs
• emit events

Example:

```
UserCardComponent
```

Example structure:

```
DashboardContainer
   ↓
UserCardComponent
   ↓
ChartComponent
```

Benefits:

• reusable UI
• clear separation
• easier testing

# COMPONENT LIFECYCLE

Angular components go through a lifecycle.

Lifecycle phases:

```
Creation
Initialization
Change Detection
Rendering
Destruction
```

Angular exposes lifecycle hooks.

==================================================

# IMPORTANT LIFECYCLE HOOKS

### ngOnInit

Runs after component initialization.

Example:

```ts
ngOnInit() {
 this.loadUsers();
}
```

Used for:

• API calls
• initialization logic

---

### ngOnChanges

Runs when input values change.

Example:

```ts
ngOnChanges(changes: SimpleChanges) {
 console.log(changes);
}
```

---

### ngAfterViewInit

Runs after view initialization.

Useful for:

• DOM access
• third-party libraries

---

### ngOnDestroy

Runs before component destruction.

Used for:

• unsubscribing from observables
• cleanup

Example:

```ts
ngOnDestroy() {
 this.subscription.unsubscribe();
}
```

### Mental Anchor

> Lifecycle hooks allow components to react to **runtime events**.

# COMPONENT RENDERING FLOW

Angular rendering works like this:

```
Component Created
     ↓
Template Parsed
     ↓
Bindings Evaluated
     ↓
DOM Rendered
     ↓
Change Detection Updates DOM
```

Example template binding:

```html
<h2>{{user.name}}</h2>
```

Angular updates DOM whenever data changes.

# STANDALONE COMPONENTS (MODERN ANGULAR)

Angular now supports standalone components.

Example:

```ts
@Component({
 standalone: true,
 selector: 'app-user',
 template: `<h1>User</h1>`
})
export class UserComponent {}
```

Benefits:

• no NgModule required
• simpler architecture
• better tree shaking

Standalone components are becoming **standard in modern Angular**.

# COMPONENT DESIGN PRINCIPLES

Senior engineers follow strict rules.

✔ Components should be **small**
✔ UI logic separate from business logic
✔ Prefer services for data
✔ Avoid large templates
✔ Avoid deep component nesting

Bad example:

```
DashboardComponent
(2000 lines)
```

Good architecture:

```
DashboardContainer
   ├── HeaderWidget
   ├── StatsWidget
   ├── ChartWidget
   └── ActivityWidget
```

> Angular applications are component trees
