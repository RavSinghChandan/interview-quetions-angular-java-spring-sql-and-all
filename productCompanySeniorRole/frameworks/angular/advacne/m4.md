# CORE IDEA: TEMPLATES ARE NOT HTML

Many developers believe Angular templates are simply HTML.

This is incorrect.

Angular templates are actually **compiled UI instructions**.

Example template:

```html
<h2>{{user.name}}</h2>
```

Angular does not render this directly.

Instead, Angular compiles it into **JavaScript rendering instructions**.

Conceptual example:

```text
createElement("h2")
bindText(user.name)
```

### Mental Anchor

> Angular templates are **compiled into rendering instructions** executed by the Angular runtime.

==================================================

# THE ANGULAR TEMPLATE COMPILER

Angular has its own compiler.

It transforms templates into optimized JavaScript.

Process:

```text
Template HTML
     ↓
Angular Compiler
     ↓
Generated Rendering Code
     ↓
Browser DOM
```

Example template:

```html
<div>{{title}}</div>
```

Conceptual compiled output:

```javascript
ɵɵelementStart(0, "div");
ɵɵtext(1);
ɵɵelementEnd();
```

Then binding:

```javascript
ɵɵtextBinding(1, ctx.title);
```

This is part of Angular's **Ivy rendering engine**.

# WHAT IS A VIEW IN ANGULAR

Angular does not render components directly.

It renders **views**.

A view is:

```text
Template Instance + Data Context
```

Example component template:

```html
<h1>{{title}}</h1>
```

Angular creates a **view instance** for it.

View structure:

```text
View
 ├── DOM nodes
 ├── binding instructions
 └── change detection metadata
```

### Mental Anchor

> Angular renders **views**, not components.

==================================================

# TYPES OF VIEWS

Angular has two main types.

---

# 1. HOST VIEW

Created when Angular instantiates a component.

Example:

```ts
@Component({
 selector: 'app-user',
 template: `<h1>User</h1>`
})
```

Angular creates a **host view** to host the component.

Structure:

```text
Host Element
    ↓
Component View
```

Example DOM:

```html
<app-user>
   <h1>User</h1>
</app-user>
```

---

# 2. EMBEDDED VIEW

Created by structural directives.

Example:

```html
<div *ngIf="isVisible">Hello</div>
```

Angular internally transforms it to:

```html
<ng-template [ngIf]="isVisible">
   <div>Hello</div>
</ng-template>
```

Angular dynamically creates or destroys the **embedded view**.

==================================================

# STRUCTURAL DIRECTIVES (DEEP UNDERSTANDING)

Structural directives modify the **DOM structure**.

Examples:

```text
*ngIf
*ngFor
*ngSwitch
```

Example:

```html
<div *ngIf="isAdmin">Admin Panel</div>
```

Angular internally converts it to:

```html
<ng-template [ngIf]="isAdmin">
   <div>Admin Panel</div>
</ng-template>
```

The directive controls whether the view exists.

==================================================

# VIEW CONTAINER

Structural directives use a **ViewContainerRef**.

It acts as a placeholder for dynamic views.

Example conceptual structure:

```text
ViewContainer
   ├── View 1
   ├── View 2
   └── View 3
```

Example: `*ngFor`

```html
<li *ngFor="let user of users">
  {{user.name}}
</li>
```

Angular creates one embedded view per item.

```text
ViewContainer
   ├── UserView 1
   ├── UserView 2
   ├── UserView 3
```

==================================================

# TEMPLATE REFERENCE VARIABLES

Templates can expose variables.

Example:

```html
<input #username>
<button (click)="print(username.value)">
```

Angular assigns the DOM element to the variable.

This allows interaction between template and component.

==================================================

# TEMPLATE CONTEXT

Every template runs with a **context object**.

Example:

```html
{{user.name}}
```

Here:

```text
context = component instance
```

Example:

```ts
export class UserComponent {
 user = { name: "Chandan" };
}
```

Angular resolves:

```text
ctx.user.name
```

# TEMPLATE BINDING TYPES

Angular templates support multiple binding types.

---

### Interpolation

```html
{{title}}
```

---

### Property Binding

```html
<img [src]="imageUrl">
```

---

### Event Binding

```html
<button (click)="save()">
```

---

### Two-way Binding

```html
<input [(ngModel)]="username">
```

Angular combines:

```text
Property Binding + Event Binding
```

# TEMPLATE PERFORMANCE CONSIDERATIONS

Templates execute frequently during **change detection**.

Bad template patterns cause performance issues.

Bad example:

```html
{{calculateTotal()}}
```

This method runs **every change detection cycle**.

Better:

```html
{{total}}
```

Compute in component instead.

==================================================

# PURE VS IMPURE PIPES

Angular pipes transform template values.

Example:

```html
{{price | currency}}
```

Pipes can be:

### Pure Pipes

Run only when inputs change.

### Impure Pipes

Run every change detection cycle.

Impure pipes can hurt performance.

Example dashboard template:

```html
<tr *ngFor="let trade of trades">
   <td>{{trade.symbol}}</td>
```

```text
ViewContainer
```

When data changes:

Angular updates **only affected views**.
