# CORE IDEA: WHAT THE ANGULAR ROUTER DOES

Angular applications are **Single Page Applications (SPA)**.

This means:

```text
Browser URL changes
       ↓
Angular router intercepts navigation
       ↓
Component rendered without full page reload
```

Example URL:

```
/dashboard
/users
/trades/123
```

Angular maps these URLs to **components**.

### Mental Anchor

> Angular router maps **URL → Component View**.

==================================================

# BASIC ROUTER CONFIGURATION

Routes are defined using a configuration object.

Example:

```ts
const routes: Routes = [
 {
   path: 'dashboard',
   component: DashboardComponent
 },
 {
   path: 'users',
   component: UserListComponent
 }
];
```

Register router:

```ts
@NgModule({
 imports: [RouterModule.forRoot(routes)],
 exports: [RouterModule]
})
export class AppRoutingModule {}
```

Root template:

```html
<router-outlet></router-outlet>
```

Router outlet acts as a **placeholder for routed components**.

==================================================

# ROUTER NAVIGATION FLOW

When navigation occurs:

Example:

```
/users
```

Angular performs the following steps:

```text
URL change detected
      ↓
Router parses URL
      ↓
Route configuration matched
      ↓
Route guards evaluated
      ↓
Resolvers executed
      ↓
Component instantiated
      ↓
Component rendered in router outlet
```

This process is called **router navigation lifecycle**.

# ROUTE PARAMETERS

Routes can contain parameters.

Example:

```ts
{
 path: 'users/:id',
 component: UserDetailComponent
}
```

URL:

```
/users/101
```

Access parameter:

```ts
constructor(private route: ActivatedRoute) {}

ngOnInit() {
 const id = this.route.snapshot.params['id'];
}
```

For reactive approach:

```ts
this.route.params.subscribe(params => {
 console.log(params['id']);
});
```

==================================================

# QUERY PARAMETERS

Query parameters appear after `?`.

Example URL:

```
/users?page=2
```

Access them:

```ts
this.route.queryParams.subscribe(params => {
 console.log(params['page']);
});
```

Used for:

• filtering
• pagination
• search queries

# ROUTE GUARDS (SECURITY & CONTROL)

Route guards control navigation.

Types of guards:

```text
CanActivate
CanDeactivate
CanLoad
Resolve
```

Example authentication guard:

```ts
@Injectable({ providedIn: 'root' })
export class AuthGuard implements CanActivate {

 constructor(private authService: AuthService) {}

 canActivate(): boolean {
   return this.authService.isLoggedIn();
 }

}
```

Route configuration:

```ts
{
 path: 'admin',
 component: AdminComponent,
 canActivate: [AuthGuard]
}
```

Angular checks guard before navigation.

==================================================

# ROUTE RESOLVERS

Resolvers fetch data **before navigation completes**.

Example:

```ts
@Injectable({ providedIn: 'root' })
export class UserResolver implements Resolve<any> {

 constructor(private userService: UserService) {}

 resolve() {
   return this.userService.getUsers();
 }

}
```

Route configuration:

```ts
{
 path: 'users',
 component: UserListComponent,
 resolve: { users: UserResolver }
}
```

Component receives resolved data.

# LAZY LOADING (VERY IMPORTANT)

Large Angular applications must reduce **initial bundle size**.

Lazy loading loads modules **only when needed**.

Example route:

```ts
{
 path: 'admin',
 loadChildren: () =>
   import('./admin/admin.module')
     .then(m => m.AdminModule)
}
```

This means:

Admin module code is loaded **only when `/admin` route is visited**.

### Mental Anchor

> Lazy loading reduces **initial load time**.

# ROUTER MODULE ARCHITECTURE

Large Angular applications structure routes like this:

```text
App Routing
   ↓
Feature Modules
   ↓
Child Routes
```

Example structure:

```text
AppRoutingModule
│
├── DashboardModule
│
├── TradingModule
│
└── AdminModule
```

Each module has its own routing configuration.

# CHILD ROUTES

Routes can be nested.

Example:

```ts
{
 path: 'dashboard',
 component: DashboardComponent,
 children: [
   {
     path: 'trades',
     component: TradesComponent
   },
   {
     path: 'portfolio',
     component: PortfolioComponent
   }
 ]
}
```

Template:

```html
<router-outlet></router-outlet>
```

Nested router outlets enable complex layouts.

# PROGRAMMATIC NAVIGATION

Angular allows navigation through code.

Example:

```ts
constructor(private router: Router) {}

goToDashboard() {
 this.router.navigate(['/dashboard']);
}
```

With parameters:

```ts
this.router.navigate(['/users', 101]);
```

With query parameters:

```ts
this.router.navigate(['/users'], {
 queryParams: { page: 2 }
});
```

# ROUTER PERFORMANCE CONSIDERATIONS

Routing affects performance.

Common issues:

• loading too many modules initially
• large bundle size
• unnecessary guards/resolvers

Best practices:

✔ use lazy loading
✔ split features into modules
✔ avoid heavy resolver logic
