# CORE IDEA: ANGULAR HTTP ARCHITECTURE

Angular communicates with backend APIs using:

```text
HttpClient
```

But internally the architecture is layered:

```
Component
   ↓
Service
   ↓
HttpClient
   ↓
Interceptor Chain
   ↓
HttpBackend
   ↓
Browser HTTP API
```

### Mental Anchor

> Angular HTTP requests pass through a **pipeline of interceptors before reaching the server**.

# HTTPCLIENT BASICS

Angular HTTP calls are made using `HttpClient`.

Example:

```ts
constructor(private http: HttpClient) {}

getUsers() {
 return this.http.get('/api/users');
}
```

This returns an **Observable**.

Usage:

```ts
this.userService.getUsers()
 .subscribe(users => {
   this.users = users;
 });
```

### Important

Angular HTTP is **fully reactive** using RxJS.

# HTTP REQUEST OBJECT

Angular internally represents requests as immutable objects.

Example:

```
HttpRequest
```

Structure conceptually:

```
HttpRequest
 ├── URL
 ├── Method
 ├── Headers
 ├── Body
 └── Params
```

Because requests are immutable, interceptors must **clone them before modification**.

Example:

```ts
req.clone({
 headers: req.headers.set('Authorization', token)
})
```

==================================================

# INTERCEPTOR PIPELINE (VERY IMPORTANT)

Angular HTTP interceptors behave like **middleware**.

They intercept every HTTP request.

Pipeline:

```
Request
  ↓
Interceptor 1
  ↓
Interceptor 2
  ↓
Interceptor 3
  ↓
HttpBackend
  ↓
Server
```

Response flows back through the same chain.

```
Server Response
      ↑
Interceptor 3
      ↑
Interceptor 2
      ↑
Interceptor 1
      ↑
Component
```

# CREATING AN HTTP INTERCEPTOR

Example authentication interceptor.

```ts
@Injectable()
export class AuthInterceptor implements HttpInterceptor {

 intercept(req: HttpRequest<any>, next: HttpHandler) {

   const token = localStorage.getItem("token");

   const authReq = req.clone({
     setHeaders: {
       Authorization: `Bearer ${token}`
     }
   });

   return next.handle(authReq);

 }

}
```

Register interceptor:

```ts
providers: [
 {
   provide: HTTP_INTERCEPTORS,
   useClass: AuthInterceptor,
   multi: true
 }
]
```

`multi: true` allows multiple interceptors.

# ERROR HANDLING ARCHITECTURE

Angular HTTP errors return an `HttpErrorResponse`.

Example:

```ts
this.http.get('/api/users')
 .subscribe({
   next: data => console.log(data),
   error: err => console.error(err)
 });
```

Global error handling can be done via interceptors.

Example:

```ts
return next.handle(req).pipe(
 catchError(error => {
   console.error("API error", error);
   return throwError(() => error);
 })
);
```

==================================================

# RETRY LOGIC

Angular allows retry logic using RxJS.

Example:

```ts
return this.http.get('/api/data')
 .pipe(
   retry(3)
 );
```

This retries failed requests up to 3 times.

Often used for:

• network failures
• temporary backend issues

Common enterprise authentication pattern.

Flow:

```

   ↓

   ↓
Token stored in browser
   ↓
Interceptor attaches token to requests
   ↓
Backend validates token
```

Example header:

```
Authorization: Bearer <JWT_TOKEN>
```

# REQUEST CANCELLATION

Angular allows cancellation of HTTP requests.

Example using RxJS:

```ts
takeUntil(this.destroy$)
```

Common scenario:

User navigates away before request completes.

Cancellation prevents:

• memory leaks
• unnecessary network activity

# INTERVIEW PUNCHLINE (STRONG)

> Angular's HttpClient communicates with backend APIs through a reactive pipeline where requests pass through interceptor chains before reaching the server.
