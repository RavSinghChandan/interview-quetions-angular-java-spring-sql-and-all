# CORE IDEA: WHAT REACTIVE PROGRAMMING IS

Traditional programming handles values like this:

```text
value → process → result
```

Reactive programming handles **streams of values**.

Example:

```text id="rxs1"
value1
value2
value3
value4
```

These values arrive over time.

Reactive programming processes them as a **data stream**.

### Mental Anchor

> RxJS treats data as **streams instead of single values**.

==================================================

# WHAT IS AN OBSERVABLE

Observable is the **core primitive in RxJS**.

Definition:

```text id="rxs2"
Observable = stream of asynchronous values over time
```

Example observable:

```ts
import { Observable } from 'rxjs';

const stream = new Observable(observer => {

 observer.next(1);
 observer.next(2);
 observer.next(3);

 observer.complete();

});
```

Subscriber:

```ts
stream.subscribe(value => {
 console.log(value);
});
```

Output:

```
1
2
3
```

==================================================

# OBSERVABLE LIFECYCLE

An observable has three signals:

```text id="rxs3"
next(value)
error(err)
complete()
```

Example:

```ts
observable.subscribe({
 next: value => console.log(value),
 error: err => console.error(err),
 complete: () => console.log("done")
});
```

# HOT VS COLD OBSERVABLES

This is a **common interview question**.

---

# COLD OBSERVABLE

Cold observables start producing values **when subscribed**.

Example:

```ts
const observable = this.http.get('/api/users');
```

Each subscription triggers a **new request**.

```text id="rxs5"
Subscriber A → API call
Subscriber B → API call
```

Cold streams are **lazy**.

---

# HOT OBSERVABLE

Hot observables produce values **independent of subscribers**.

Example:

```ts
const subject = new Subject<number>();
```

Subscribers receive values from a **shared stream**.

```text id="rxs6"
Event Source
     ↓
Subject
     ↓
Subscriber A
Subscriber B
```

==================================================

# SUBJECTS

Subjects are both:

```text id="rxs7"
Observable + Observer
```

Meaning they can:

• emit values
• allow subscriptions

Example:

```ts
import { Subject } from 'rxjs';

const subject = new Subject<number>();

subject.subscribe(v => console.log("A:", v));
subject.subscribe(v => console.log("B:", v));

subject.next(1);
subject.next(2);
```

Output:

```
A:1
B:1
A:2
B:2
```

==================================================

# BEHAVIORSUBJECT

Very common in Angular services.

BehaviorSubject stores the **latest value**.

Example:

```ts
import { BehaviorSubject } from 'rxjs';

const user$ = new BehaviorSubject(null);
```

Emit value:

```ts
user$.next({name: "Chandan"});
```

New subscribers receive **current value immediately**.

Example:

```text id="rxs8"
Subscriber joins
↓
Receives latest state
```

Used heavily for:

• application state
• UI state sharing

==================================================

# REPLAYSUBJECT

ReplaySubject replays previous values.

Example:

```ts
const replay = new ReplaySubject(3);
```

Stores last **3 emitted values**.

New subscribers receive them.

Used for:

• caching streams
• debugging flows

# RXJS OPERATORS

Operators transform observable streams.

Example:

```ts
this.http.get('/api/users')
 .pipe(
   map(users => users.filter(u => u.active))
 )
```

Common operators:

```text id="rxs9"
map
filter
switchMap
mergeMap
concatMap
debounceTime
```

Operators allow building **data pipelines**.

==================================================

# SWITCHMAP VS MERGEMAP (INTERVIEW FAVORITE)

Example scenario: search input.

User types quickly.

Bad approach:

```ts
mergeMap
```

All requests execute.

Better approach:

```ts
switchMap
```

Previous request is cancelled.

Example:

```ts
search$
 .pipe(
   debounceTime(300),
   switchMap(query => this.http.get(`/search?q=${query}`))
 )
```

==================================================

# MEMORY LEAKS IN RXJS

Common Angular problem.

Example:

```ts
this.service.user$.subscribe(...)
```

If subscription is not cleaned up:

• component destroyed
• observable still active

Memory leak occurs.

==================================================

# PREVENTING MEMORY LEAKS

Common pattern:

```ts
destroy$ = new Subject();
```

Subscription:

```ts
this.service.user$
 .pipe(takeUntil(this.destroy$))
 .subscribe();
```

Cleanup:

```ts
ngOnDestroy() {
 this.destroy$.next();
 this.destroy$.complete();
}
```

==================================================

# ASYNC PIPE (BEST PRACTICE)

Angular provides `async` pipe.

Example:

```html
<div *ngIf="user$ | async as user">
 {{user.name}}
</div>
```

Benefits:

• automatic subscription
• automatic unsubscription

Avoids memory leaks.
