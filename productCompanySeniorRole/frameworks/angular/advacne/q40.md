# 1. What happens when an Angular application starts?

**Answer**

Angular bootstraps from `main.ts`.

Flow:

```
Browser loads index.html
↓
Angular runtime initializes
↓
platformBrowserDynamic() creates platform
↓
AppModule bootstrapped
↓
Root injector created
↓
Root component instantiated
↓
Component tree rendered
↓
Change detection begins
```

Key insight:

Angular startup builds the **dependency graph + component tree**, then activates the **change detection engine**.

# 2. What is Angular’s Dependency Injection system?

Angular DI is a **hierarchical injector system** that manages service creation and dependency wiring.

Hierarchy:

```
Root Injector
   ↓
Module Injector
   ↓
Component Injector
```

Resolution algorithm:

```
Component Injector
↓
Parent Injector
↓
Root Injector
```

If no provider exists → `NullInjectorError`.

DI enables:

* loose coupling
* singleton management
* testability

# 3. Difference between `providedIn: 'root'` and module providers?

### providedIn: 'root'

```
@Injectable({ providedIn: 'root' })
```

• singleton instance
• tree-shakable
• global service

### module provider

```
@NgModule({
 providers: [UserService]
})
```

• scoped to module injector
• lazy modules create new instances

# 4. What is Angular Change Detection?

Change detection synchronizes **application state with the DOM**.

Flow:

```
Event occurs
↓
Zone.js detects async event
↓
Angular triggers change detection
↓
Component tree checked
↓
Bindings re-evaluated
↓
DOM updated
```

Angular checks components **top-down**.

---

# 5. Why is `OnPush` faster?

Default strategy:

```
Every async event
↓
Entire component tree checked
```

OnPush strategy:

```
Only check when:
Input reference changes
Event occurs
Observable emits
```

This reduces **unnecessary change detection cycles**.

---

# 6. What role does Zone.js play in Angular?

Zone.js patches async APIs like:

```
setTimeout
Promise
DOM events
XHR
```

This allows Angular to know:

```
Async event occurred
↓
Run change detection
```

Without Zone.js Angular wouldn't know when to update UI.

# 7. How does Angular render templates internally?

Angular templates are compiled into **rendering instructions**.

Example template:

```html
<h1>{{title}}</h1>
```

Compiled output (conceptual):

```
ɵɵelementStart("h1")
ɵɵtextBinding(title)
ɵɵelementEnd()
```

These instructions are executed by the **Ivy rendering engine**.

# 10. What is lazy loading?

Lazy loading loads modules **only when the route is visited**.

Example:

```ts
{
 path: 'admin',
 loadChildren: () =>
   import('./admin/admin.module')
}
```

Benefits:

* smaller initial bundle
* faster startup
* modular architecture

---

# 11. What are Angular interceptors?

Interceptors act as **middleware for HTTP requests**.

Pipeline:

```
Request
↓
AuthInterceptor
↓
LoggingInterceptor
↓
RetryInterceptor
↓
Server
```

Response flows back through the same chain.

Use cases:

* authentication
* logging
* error handling
* retries

---

# 12. Why are HTTP requests immutable?

Angular uses immutable `HttpRequest`.

Modification requires cloning.

Example:

```ts
req.clone({
 setHeaders: { Authorization: token }
})
```

Benefits:

* predictable behavior
* interceptor safety

# 13. What is an Observable?

Observable = **stream of values over time**.

Example:

```
1 → 2 → 3 → 4
```

Observables allow Angular to handle:

* HTTP responses
* router events
* form updates
* WebSocket streams

---

# 14. Difference between hot and cold observables?

### Cold observable

Execution starts per subscriber.

Example:

```
HttpClient.get()
```

Each subscription triggers new request.

### Hot observable

Shared event stream.

Example:

```
Subject
WebSocket
```

---

# 15. What is a BehaviorSubject?

BehaviorSubject stores **latest emitted value**.

Example:

```ts
const user$ = new BehaviorSubject(null);
```

New subscribers immediately receive current state.

Used for:

* application state
* shared data streams

Architecture:

```

↓
Reducer
↓
Store
↓
Selector

```
