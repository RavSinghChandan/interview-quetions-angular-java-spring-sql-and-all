# 2Ô∏è‚É£ GOLDEN RULE

Behavior should live where data lives.

# 3Ô∏è‚É£ SRP AT SYSTEM LEVEL

- login

# 4Ô∏è‚É£ RESPONSIBILITY IDENTIFICATION METHOD

For each entity ask:

# 5Ô∏è‚É£ INFORMATION EXPERT PRINCIPLE (INTERVIEW GOLD)

Assign responsibility to class that has required data.

Example:

- Order
- PaymentService
- User

Because Order has items.

---

# 6Ô∏è‚É£ CONTROLLER PRINCIPLE

System operations should be handled by controller-like class.

Controller should:
- orchestrate calls
- not contain business logic

# 7Ô∏è‚É£ CREATOR PRINCIPLE

---

# 8Ô∏è‚É£ LOW COUPLING PRINCIPLE

---

# 9Ô∏è‚É£ HIGH COHESION PRINCIPLE

Each class should do closely related tasks.

- sends email
- logs metrics
- writes DB

# 1Ô∏è‚É£4Ô∏è‚É£ METHOD PLACEMENT RULE
